## Roadmap Extraction

- Baseline roadmap outcome: implement the reference model that represents storage, datasource structure, and lazy IO requests without loading payloads.
- Prerequisites: Stage 1 naming/metadata/split vocabulary and Stage 2 loaded runtime core are expected to be available. Current code shows `rphys.data` runtime contracts are implemented and `rphys.io`/`rphys.datasources` are still empty public surfaces.
- Prior or adjacent roadmap links: Stage 2 supplies loaded `FieldValue`, `Sample`, `Batch`, contracts, and collation; Stage 4 will add codecs, load/save contexts, lazy `SampleField`, and `SampleBuilder`; Stage 5 will add datasource adapters, views, filters, groups, splits, indexes, and index manifests.
- Primary feature docs: none found; `docs/roadmap.md` is the canonical source.
- Deferred or out-of-scope roadmap work: codecs, codec registry, load/save contexts, lazy sample fields, `SampleBuilder`, datasource scanning/builders/filters/splits/index manifests, synthetic datasource adapters, operations, transforms, export/save ops, training loaders/cache, workflow/artifact runtime, spatial slices, seconds-based slices, and multi-member index items.
- Compatibility obligations: preserve lightweight imports; keep `FieldRef` free of runtime role, temporal slice, loaded tensors, and open handles; keep `IndexItem` pure lazy IO; fail loudly for unsupported slices; keep cross-field temporal alignment explicit rather than inferred from matching numeric slices; avoid generic workflow/artifact abstractions.
- Public surfaces or durable artifacts likely affected: `rphys.io` public exports for `ResourceRef`, `FieldRef`, `FieldIndex`, `TemporalIndexSlice`, and `FieldView`; `rphys.datasources` exports or submodules for `DataSourceRef`, `RecordRef`, `IndexItem`, and `DataSourceSchema`; `src/rphys/errors.py` if concrete IO/datasource/slice diagnostics are required; package import-boundary tests; source-mirrored unit tests; public contract tests and docstrings. Durable serialization is likely relevant for refs/views because the roadmap says refs and views serialize without open handles or loaded arrays, but manifest/index codecs are deferred.

## Overview

- What this stage covers: lazy storage/resource references, datasource and record references, field references, field views, the initial temporal index abstraction, and role-qualified `IndexItem` mappings consumed later by sample builders.
- Why it exists: downstream codecs, sample builders, datasource indexes, export, and training adapters need a dependency-light way to describe what to load without loading payloads or mixing in transforms, methods, export logic, or workflow state.
- Primary outcomes: a clear `DataSourceRef -> RecordRef -> FieldRef -> FieldView -> IndexItem` chain; serializable lazy refs/views with no open handles or loaded arrays; field-native temporal slicing through an initial `TemporalIndexSlice`; and explicit separation between intrinsic field identity, runtime role, and lazy access behavior.
- Non-goals: payload loading, codecs, probing, save/export behavior, lazy `SampleField`s, sample construction, datasource scanning/filtering/splitting/index manifests, transforms/augmentations, methods/models, training loader/cache behavior, artifact refs, stages, workflow runtime, spatial slices, seconds-based slicing, implicit cross-field alignment, and multi-member contrastive index items.
- Related feature docs: none found.
- Impacted repo areas: `src/rphys/io/`; `src/rphys/datasources/`; `src/rphys/errors.py`; `tests/package/`; `tests/unit/rphys/io/` if added later; `tests/unit/rphys/datasources/` if added later; `tests/contracts/`; docstrings and glossary/roadmap references if public wording needs alignment.
- Current state: `rphys.io` and `rphys.datasources` import with empty `__all__`; package tests expect deferred package homes to be empty except `rphys.data`; Stage 1/2 data contracts are implemented; no lazy IO classes, datasource refs, field indexes, field views, index items, or lazy IO tests exist.
- Key uncertainty: the functionality, design boundary, validation strategy, phase sketch, plan quality gate, and implementation-plan draft are locked; maintainer approval of the implementation plan remains pending before execution.
- User clarification questions and resolved answers: no initial clarification required; roadmap intent, target audience, and planning priority are evidence-backed by `docs/roadmap.md`, `docs/GLOSSARY.md`, and Stage 2 planning artifacts.
- Planning priority or optimization target: define a small, dependency-free lazy reference model that composes with existing runtime data contracts and later codec/datasource stages without prematurely freezing full datasource builder, manifest, cache, or sample-builder APIs.

## Behavior Confirmation

- Included behavior: Stage 3 includes dependency-free lazy descriptors for addressable storage targets (`ResourceRef`), complete logical fields (`FieldRef`), field indexes and initial field-native temporal slices (`FieldIndex`, `TemporalIndexSlice`), lazy load requests (`FieldView`), datasource/view identity (`DataSourceRef`), stable pre-window records with field presence and leakage-sensitive metadata (`RecordRef`), role-qualified lazy field mappings (`IndexItem`), a minimal datasource field/schema descriptor (`DataSourceSchema`), and the public import/error boundary needed to make those objects usable.
- Default behavior: descriptors remain serializable through primitive data without open handles or loaded arrays; `FieldRef` stays free of runtime role and temporal slice; `FieldView` carries optional access behavior; `IndexItem` maps `FieldLocator` to `FieldView`; temporal slices are index-based, field-native, half-open, non-negative, and positive-step; equal numeric slices on different fields do not imply alignment.
- Failure behavior: invalid resource, field, datasource, record, index-item, schema, or slice structure should fail loudly with typed diagnostics under the existing IO, datasource, field, or slice error families; unsupported slice behavior such as non-supported `step` values must raise rather than silently full-loading, truncating, padding, or converting units.
- Unsupported behavior: no payload loading, probing, codec registry, load/save contexts, lazy `SampleField`, `SampleBuilder`, datasource scanning/building/filtering/splitting, index manifests, transforms, augmentations, methods, export, training adapters, caches, workflow/artifact refs, seconds-based slices, spatial slices, implicit temporal alignment, nested/multi-member index items, or root-level convenience re-exports.
- Resume/interruption behavior: no runtime resume semantics are introduced; the planning artifact records this locked functionality baseline and continues to design proposal without a context reset.
- Downstream implications: Stage 4 codecs and `SampleBuilder` can consume `FieldView` and `IndexItem` without accepting raw paths, runtime samples, or transform logic; Stage 5 datasource builders and manifests can build around `DataSourceRef`, `RecordRef`, `DataSourceSchema`, and `IndexItem` without reworking Stage 3 identities.
- Explicit deferrals: datasource-index item identity/fingerprints, expected-versus-observed schema distinctions, descriptor fingerprint semantics, codecs/sample building, datasource builders/manifests, seconds/spatial slicing, multi-member index items, export, training, and workflow artifacts are deferred to later roadmap stages.
- Why this behavior is locked: FQ-1 through FQ-8 are repo-resolved by `docs/roadmap.md`, `docs/GLOSSARY.md`, Stage 2 boundaries, current package homes, error taxonomy, and package/import tests; no high-impact functionality blocker or maintainer decision packet remains before design work.

## Proposed Implementation Shape

- Likely modules or packages: add focused dependency-free modules under `rphys.io` and `rphys.datasources`, plus concrete Stage 3 errors in `rphys.errors`. Recommended public submodule shape is `rphys.io.resources` for `ResourceRef`, `rphys.io.fields` for `FieldRef` and `FieldView`, `rphys.io.indexes` for `FieldIndex` and `TemporalIndexSlice`, `rphys.datasources.refs` for `DataSourceRef` and `RecordRef`, `rphys.datasources.schemas` for `DataSourceSchema`, and `rphys.datasources.index_items` for `IndexItem`. Re-export implemented names from `rphys.io` and `rphys.datasources`; keep root `rphys` without convenience re-exports. This exact public import-path shape is approval-worthy.
- Likely public classes/functions/protocols: dependency-free slotted descriptor classes for `ResourceRef`, `FieldRef`, `TemporalIndexSlice`, `FieldView`, `DataSourceRef`, `RecordRef`, `DataSourceSchema`, and `IndexItem`; a narrow public `FieldIndex` protocol or base interface for future index descriptors; explicit `to_dict()`/`from_dict()` descriptor serialization methods if approved; concrete Stage 3 typed errors for invalid refs, invalid slices/indexes, invalid datasource descriptors, invalid schemas, invalid index items, and unsupported index behavior.
- Likely internal helpers: private coercion and validation helpers for non-empty strings, primitive JSON-like values, metadata-key mappings, resource tuples, field-view dictionaries, and descriptor serialization dispatch. These helpers should stay private, side-effect-free, and local to the owning package/module rather than becoming registries or global validators.
- Data flow: `ResourceRef` describes an addressable storage target through URI/protocol/storage options; the URI may name the most precise target available, such as a file, shard, array, stream, archive entry, or URI fragment, but Stage 3 does not parse or normalize internal URI meaning. `FieldRef` binds one complete logical `DataKey`/optional schema to one or more `ResourceRef`s; `TemporalIndexSlice` describes field-native half-open access; `FieldView` pairs a complete `FieldRef` with an optional `FieldIndex`; `DataSourceRef` names a datasource or datasource view with mandatory datasource identity; `DataSourceSchema` declares non-empty `DataKey -> FieldSpec` field structure without validation evidence; `RecordRef` points to a datasource, stable record identity, non-empty field presence by intrinsic `DataKey`, and leakage-sensitive metadata; `IndexItem` maps role-qualified `FieldLocator`s to `FieldView`s and carries mandatory `RecordRef` provenance without owning stable datasource-index item identity.
- Dependency direction: `rphys.io` may depend on `rphys.data.keys`, `rphys.data.schemas`, and private local helpers, but not on `rphys.datasources`, codecs, sample builders, runtime `Sample`, operations, training, or workflow packages. `rphys.datasources` may depend on `rphys.data` vocabulary/runtime `FieldSpec` for `DataSourceSchema`, and on `rphys.io` descriptors, but `rphys.io` must not depend back on datasources. `rphys.errors` remains the shared leaf for typed diagnostics. No heavy optional dependency is introduced.
- Extension points: future resource protocols and field indexes extend by constructing importable Python objects with primitive descriptor serialization, not by adding a Stage 3 registry. Future codecs consume `FieldView`; future datasource adapters/builders emit `DataSourceRef`, `RecordRef`, `FieldRef`, `FieldView`, and `IndexItem`; future operation/training/export code consumes later runtime layers and must not be embedded in these descriptors.
- Compatibility constraints: preserve lightweight imports; no root re-exports; no open handles, existence checks, probing, payload arrays, pickle-only contracts, hidden URI parsing, descriptor fingerprints, validation evidence/status, implicit temporal alignment, seconds/spatial slicing, transforms, sample building, datasource scanning, manifest codecs, datasource-index item identity, `item_id`, or workflow/artifact state.
