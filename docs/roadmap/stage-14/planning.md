## Roadmap Extraction

- Baseline roadmap outcome: make the public object model difficult to regress
  by consolidating private synthetic fixtures, contract helpers, integration
  flows, smoke tests, and public API/import checks under repository-level
  tests.
- Prerequisites: Stage 5 datasource/index/sample-builder synthetic flow;
  Stage 6/7 operation and pipeline contracts; Stage 8 export/save and derived
  datasource round trips; Stage 9 sample source/cache/prepared/torch adapter
  contracts; Stage 10 method prediction; Stage 11 collections, metrics,
  losses, objectives; Stage 12 learner/trainer; Stage 13 Sample/Batch-native
  prediction fields, uncollation, sample artifact export/reload, runtime
  collections, metric operations, visualization/report records, and recipe
  behavior for the complete root smoke endpoint.
- Prior or adjacent roadmap links: Stage 13 defines the final Sample/Batch,
  artifact handoff, collection/metric, visualization/report, and recipe
  surfaces consumed by the smoke path; Stage 15 later profiles training/data
  paths but should not change Stage 14 smoke semantics.
- Primary feature docs: none found under `docs/features`.
- Deferred or out-of-scope roadmap work: public testing-helper package,
  real datasource adapters, raw datasets, acceptance checks requiring local
  data/hardware/network/GPU, profiling optimization, concrete algorithms,
  broad fixture registries, workflow runtime, project config, artifact store,
  hidden plotting, dataframe dependencies, and optional heavy dependencies in
  default imports.
- Compatibility obligations: testing support remains private under
  `tests/support`; synthetic fixtures are deterministic, tiny, license-safe,
  and CPU-only; smoke paths use the same loader path as ordinary code; refs and
  manifests serialize without loaded arrays/open handles; field roles,
  selectors, views, materialization, cache keys, and public imports stay
  explicit and fail-loud.
- Public surfaces or durable artifacts likely affected: no new public package
  surface by default. Durable artifacts likely include private test fixtures,
  contract helper functions/classes under `tests/support`, integration/e2e
  smoke tests, package/import-boundary assertions, and tiny manifest fixtures
  stored as test data only if the design pass approves concrete golden files.

## Overview

- What this stage covers: private test-only synthetic fixture catalog; edge
  variant fixtures; reusable contract helpers for public object-model
  invariants; integration and smoke flows over synthetic scan, filtering,
  grouping/splitting, index manifests, lazy sample building, operations,
  collation, Method/Learner returned `Batch` fields, explicit uncollation,
  sample artifact export/reload, runtime collection/metric operations,
  visualization/report records, and report recipes; package/API/import-boundary
  checks; smoke/debug/signal tier
  conventions using the same loader path.
- Why it exists: after the public object model spans the full synthetic
  vertical slice, regressions are most likely to appear at boundaries between
  packages. Stage 14 turns those boundaries into deterministic, dependency-light
  executable contracts.
- Primary outcomes: maintainers and downstream projects can validate that core
  contracts compose without real data, network access, GPUs, heavy optional
  dependencies, workflow tooling, or private project code.
- Non-goals: public `rphys.testing` helpers, real datasource implementations,
  concrete rPPG algorithms, profiling optimization, trainer/export/report
  behavior not already owned by earlier stages, rejected Stage 13 runner or
  prediction-record surfaces, acceptance datasets, optional dependency
  adapters, broad workflow/job runtime, or public fixture registries.
- Related feature docs: none found.
- Impacted repo areas: `tests/support`, `tests/contracts`,
  `tests/integration`, possibly `tests/e2e`, package tests, existing synthetic
  fixture modules, stage-specific integration tests, and docs/docstrings that
  describe test-only support and smoke tiers. Source packages should change
  only when tests expose real public-contract gaps during implementation.
- Current state: private support helpers and several stage-specific synthetic
  flows exist; `tests/e2e` and `tests/acceptance` directories are absent; the
  Stage 13 implementation plan is complete with all six phases merged.
- Key uncertainty: how broad the Stage 14 root smoke should be after consuming
  the completed Stage 13 behavior. The recommended default is to implement a
  small deterministic scan-to-report smoke, with broader signal sweeps optional.
- User clarification questions and resolved answers: none asked in this pass.
  No functionality queue item remains pending or blocked; the Stage 13 tail is
  recorded as a Phase 4 dependency with a merge/preflight check.
- Planning priority or optimization target: maximize regression resistance per
  test cost by consolidating deterministic synthetic helpers and using them
  across contract, integration, smoke, package, and import-boundary suites.

## Behavior Confirmation

- Included behavior: private deterministic synthetic fixture catalog; edge-case
  variants; private contract helpers; public object-model contract coverage;
  root synthetic smoke path with a Stage 13-aligned Sample/Batch-native tail
  gated on code-backed public Stage 13 contracts; package/API and
  import-boundary hardening; smoke/debug/signal tier conventions;
  test-support governance.
- Default behavior: support helpers live under `tests/support`; smoke tests
  use public APIs and the same loader/materialization paths as ordinary code;
  data is tiny, deterministic, license-safe, CPU-only, and generated unless a
  narrow golden manifest protects a durable public artifact format; public
  package imports remain dependency-light.
- Failure behavior: invalid fixtures and edge variants should fail through the
  same typed errors and diagnostics as production public objects; tests should
  not hide failures with skip-by-default markers unless prerequisites are truly
  optional or acceptance-only.
- Unsupported behavior: public `rphys.testing` package, real raw datasets,
  default network/GPU/hardware/optional-dependency checks, hidden fake-only
  smoke paths, workflow runtime, artifact store, dashboards, plotting
  backends, profiling optimization, and concrete algorithm catalogs.
- Resume/interruption behavior: this is a planning artifact only; no runtime
  resume behavior is locked.
- Downstream implications: downstream projects get higher confidence in the
  public object model, but they do not receive stable public fixture APIs yet.
- Explicit deferrals: public testing-helper package; real datasource smoke;
  acceptance dataset checks; Stage 15 profiling/data-path optimization; broad
  golden snapshot suite.
- Why this behavior is locked: the roadmap directly assigns Stage 14 to
  synthetic contract and smoke hardening and explicitly keeps testing support
  under repository-level `tests/`; current code/tests already establish private
  support helpers and lightweight import guardrails. The roadmap puts Stage 14
  after Stage 13, so Stage 14 consumes completed Stage 13 behavior and must not
  invent placeholder prediction, evaluation, analysis, report, runner, or record
  behavior.

## Proposed Implementation Shape

- Likely modules or packages: no new production package is proposed. Stage 14
  implementation should extend private support under `tests/support` with
  narrowly named modules such as `synthetic_catalog.py`, `synthetic_edges.py`,
  `contract_assertions.py`, `smoke_flows.py`, and targeted support-local
  fakes/codecs only when they remove duplication from consuming suites.
  Existing `tests/support/synthetic_datasources.py`,
  `tests/support/lazy_sample_builder_fixtures.py`, and
  `tests/support/synthetic_codecs.py` should either stay compatible or be
  refactored behind private helpers consumed by tests.
- Likely public classes/functions/protocols: none for Stage 14. No
  `rphys.testing`, `rphys.fixtures`, `rphys.datasets`, root re-export, public
  registry, public scenario runner, or public smoke API should be added.
  Package/import tests should only name code-backed public exports from prior
  stages and Stage 13 once Stage 13 is implemented.
- Likely private support records and helpers:
  `SyntheticFieldSpec`/scenario-style frozen dataclasses or simple factory
  functions for generated datasource/record/sample descriptions;
  deterministic payload builders for video-like frame tuples, waveform tuples,
  timestamp tuples, optional landmarks/masks/quality fields, and sidecar
  metadata; negative-variant factories for missing/short/flat/NaN/inf/
  invalid-rate/misalignment cases; assertion helpers for lazy descriptors,
  serialization/no-loaded-payload evidence, field-view access behavior,
  `SampleBuilder` probe/build subsets, operation replay evidence,
  save/export/derived datasource round trips, cache determinism, package import
  boundaries, and Stage 13 contracts when available.
- Data flow: generated fixture scenario -> `DataSourceRef` and `RecordRef`
  descriptors with URI-based `ResourceRef`s -> scan/filter/group/split/index
  through public datasource APIs -> `DataSourceIndex`/manifest round trip ->
  `SampleBuilder` and `CodecRegistry` with private synthetic codecs -> lazy
  `Sample`s and public operation pipelines -> `BatchCollater` -> trivial
  public `Method` or `Learner` returned `Batch` fields -> explicit uncollation
  to samples -> existing `SaveOperation`/derived datasource assembly -> reload
  sample artifact fields through the same index/sample path -> runtime
  collection and metric operations -> visualization/report records or report
  fields. The Stage 13 tail is planned against the completed
  Sample/Batch-native Stage 13 implementation plan.
- Dependency direction: consuming tests may import `tests.support` helpers and
  public `rphys` packages. Private support may import public `rphys` packages.
  Production `src/rphys` must not import `tests.support`; package/import tests
  should assert this. Support helpers must not reach into production-private
  modules unless a test explicitly targets private implementation behavior.
- Extension points: downstream extension remains through public `rphys`
  objects, importable callables, public operations, public codecs, public
  methods/metrics, and later Stage 13 Sample/Batch artifact, collection,
  metric, visualization, and report surfaces. Stage 14 support helpers are
  private test fixtures only. Future public testing helpers require a separate
  roadmap decision after repeated downstream need and API stability evidence.
- Compatibility constraints: deterministic, tiny, license-safe, CPU-only, and
  dependency-light fixtures; no raw datasets; no hidden fake-only loader path;
  no broad golden snapshots; no production API placeholders; root smoke tail
  does not fake Stage 13 Sample/Batch-native artifact, collection/metric,
  visualization/report, or recipe behavior, and does not reintroduce rejected
  Stage 13 public runner/record surfaces.
