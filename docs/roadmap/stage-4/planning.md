## Roadmap Extraction

- Baseline roadmap outcome: implement the bridge from `IndexItem` `FieldView`s to mutable runtime `Sample`s with lazy `SampleField` handles.
- Prerequisites: Stage 1 naming/metadata/schema/split vocabulary, Stage 2 loaded runtime core, and Stage 3 lazy descriptors/index items. Current code shows these prerequisites are implemented and tested.
- Prior or adjacent roadmap links: Stage 3 supplies `ResourceRef`, `FieldRef`, `FieldIndex`, `TemporalIndexSlice`, `FieldView`, `DataSourceRef`, `RecordRef`, `DataSourceSchema`, and `IndexItem`; Stage 5 will add datasource discovery/views/splits/index manifests; Stage 7 will rely on `SampleFields` loading only when operations access payload data; Stage 8 will save/export fields through codecs and emit derived refs/datasources.
- Primary feature docs: none found; `docs/roadmap.md` is the canonical source.
- Deferred or out-of-scope roadmap work: datasource scanning/adapters/views/filters/splits/index builders/manifests, operation foundations, SampleOps/BatchOps/pipelines, export specs/save ops/derived datasources, training adapters/loaders/caches, concrete real datasource adapters, concrete rPPG algorithms/models/losses/metrics, workflow/artifact runtime, exact full codec catalog, spatial slices, time-based slices, nested multi-view samples, multi-member `IndexItem`s, advanced cache backends, and optimized training data formats.
- Compatibility obligations: preserve lightweight imports; keep codecs scoped to `FieldView`/logical field loading and saving; keep `FieldLocator` parsing out of codecs; keep `SampleBuilder` from scanning datasources, choosing splits, applying ops, moving devices, caching implicitly, or formatting for models; ensure deterministic typed codec resolution; fail loudly for unsupported slices, ambiguous codecs, missing fields, invalid locators, missing optional dependencies, and hidden full-load fallbacks.
- Public surfaces or durable artifacts likely affected: likely new `rphys.io.codecs` public module or package for `FieldCodec`, `CodecRegistry`, `CodecCapabilities`, `CodecLoadResult`, `CodecSaveResult`, `LoadContext`, `SaveContext`, `IOContext`, and `MetadataSavePolicy`; likely focused `rphys.data.sample_fields` and `rphys.data.sample_builders` surfaces for `SampleField`, `SampleBuilder`, and `SampleBuildContext`, with package-level re-exports only if import-boundary tests support them; `src/rphys/errors.py` for exercised concrete codec/load/save/sample-builder errors; `tests/package` import/export expectations; `tests/unit/rphys/io` and `tests/unit/rphys/data`; `tests/contracts` for codec and lazy sample construction examples. Durable persisted manifests remain likely deferred, but typed load/save result objects may become durable-ish return artifacts for later export/index flows.

## Overview

- What this stage covers: field codecs, deterministic codec resolution, load/save/probe context objects, typed load/save results, lazy `SampleField` handles, `SampleBuilder` from `IndexItem` to runtime `Sample`, explicit load/error state, subset/one-field build paths, probe-without-full-load paths, and metadata save policy boundaries.
- Why it exists: later datasource indexes, operations, export, training adapters, and the synthetic vertical slice need a strict bridge between pure lazy IO descriptors and mutable loaded runtime containers without mixing datasource discovery, transforms, model formatting, caching, or workflow concerns into either layer.
- Primary outcomes: `IndexItem`s build into `Sample`s with lazy fields; codec lookup is deterministic and typed; at least one dependency-light test codec can load and save; field indexes are honored; unsupported slices and ambiguous/missing codecs fail loudly; compound logical fields can reference multiple `ResourceRef`s behind one `FieldRef`; metadata persistence remains explicit and save/export-driven.
- Non-goals: full codec catalog, real video/array library integration, datasource scanning or index construction, datasource manifests, automatic caches, export operation orchestration, derived datasource building, operation pipelines, transformations/augmentations, device movement, trainer/data-loader integration, model-specific formatting, implicit metadata saves on load, seconds/spatial slicing, hidden full-resource loading fallback, and root-level convenience exports.
- Related feature docs: none found.
- Impacted repo areas: `src/rphys/io/`; likely new `src/rphys/io/codecs.py` or `src/rphys/io/codecs/`; `src/rphys/data/`; `src/rphys/errors.py`; `tests/package/`; `tests/unit/rphys/io/`; `tests/unit/rphys/data/`; `tests/contracts/`; possibly `tests/integration/` for a small synthetic lazy-load-to-sample path if later validation planning requires it; docstrings and glossary if public wording needs alignment.
- Current state: Stage 2 runtime `Sample` stores loaded `FieldValue`s behind private entries and has accessors returning `FieldValue` or payloads. Stage 3 `IndexItem` maps `FieldLocator` to `FieldView` with mandatory `RecordRef` provenance and no runtime sample/load behavior. `rphys.io` exports Stage 3 descriptors only and explicitly has no codecs, builders, registries, or runtime payload hooks. No codec APIs, lazy fields, or sample builder APIs exist.
- Key uncertainty: resolved by design agreement. Stage 4 stores `SampleField` as a `FieldValue`-compatible lazy object in `Sample`; `field()` without validation returns the stored handle without loading, payload-demanding access may load, and loaded/error state stays inspectable on the handle.
- User clarification questions and resolved answers: pending for managing agent if later gates find product/scientific ambiguity; this context pass asked no maintainer questions. Initial evidence suggests no briefing-level ambiguity about target audience or planning priority.
- Planning priority or optimization target: prove a small dependency-light synthetic codec and lazy sample vertical slice that preserves strict field/provenance semantics and import boundaries before broader datasource, operation, export, or training behavior is planned.

## Behavior Confirmation

- Included behavior: Stage 4 includes public codec contracts and contexts/results for `FieldView` probe/load/save; deterministic typed codec resolution; a dependency-light synthetic codec that proves probe, indexed load, single-field save, and failure behavior; lazy `SampleField` handles with explicit unloaded/loaded/failed state; `SampleBuilder` construction from one `IndexItem` with all/subset/one-field requested locator modes; explicit eager loading when requested; preservation of `FieldLocator`, `DataKey`, schema, field metadata, field-native index, record provenance, item metadata, and ordered multi-resource `FieldRef.resources`; exercised concrete typed failures; intentional public imports; and contract/package/unit validation for the lazy bridge.
- Default behavior: Codecs operate on `FieldView` and codec contexts, not raw paths or role-qualified locators. Codec resolution is explicit, deterministic, and typed. `SampleBuilder` consumes an existing `IndexItem`, preserves requested locators, creates lazy handles by default, and does no datasource scanning, split choice, transform, cache, device movement, or model formatting. Probe is lightweight and must not full-load payloads. Load honors `FieldView.field_index`. Save is a single logical field operation to an explicit target description and returns typed results. Metadata persistence is explicit save/export behavior, not a side effect of load.
- Failure behavior: Missing requested locators, invalid build inputs, missing codecs, ambiguous codecs, unsupported operations or slices, missing optional dependencies, codec load/save/probe failures, and lazy field load-state errors fail loudly with typed, inspectable errors. Unsupported sliced loads must not silently full-load the resource. Load errors remain inspectable from the lazy field or result path rather than being discarded.
- Unsupported behavior: Stage 4 does not implement real video/array/image/HDF5/waveform codec catalogs; datasource discovery, filtering, splitting, indexing, or manifests; operation pipelines, transforms, augmentations, checks, or export orchestration; derived datasource creation; training/data-loader/cache/model formatting behavior; spatial slices; seconds-based slices; cross-field alignment; multi-member `IndexItem`s; nested multi-view sample structures; workflow/artifact runtimes; root convenience exports; hidden plugin discovery; implicit caches; async loading; retry policies; or mandatory heavy optional dependencies.
- Resume/interruption behavior: This planning workflow is docs-only. Runtime resume/retry semantics for lazy field loading are not introduced in Stage 4; repeated access, retained load result, and error-retention mechanics are design decisions that must preserve the locked behavior and avoid hidden reload loops or implicit caches.
- Downstream implications: Stage 5 can build datasource indexes that emit `IndexItem`s without redefining loading. Milestones 6 and 7 can trigger lazy field payload access from operations while relying on typed load failures. Milestone 8 can reuse codec save contracts for explicit export/materialization. Later training/cache/data-loader work can wrap the bridge without inheriting hidden scanning, caching, or model-formatting behavior.
- Explicit deferrals: Exact registry/resolver API shape, `CodecCapabilities` and result record fields, save-target object shape, concrete public module paths, `SampleField` storage and accessor integration with `FieldValue`/`Sample`, repeated-access semantics after successful or failed load, and any in-object loaded-value reuse are deferred to design agreement. The full codec catalog, real optional integrations, manifests, caches, export specs, operations, training adapters, and advanced slice/sample forms are deferred to later roadmap stages.
- Why this behavior is locked: The roadmap, glossary, Stage 2 runtime contracts, and Stage 3 lazy descriptor decisions converge on a narrow bridge: turn pure `IndexItem`/`FieldView` references into runtime samples through codecs without contaminating descriptors, samples, datasources, operations, export, training, or workflow layers. No functionality-level maintainer ambiguity remains; remaining questions affect implementation shape and must be resolved in design.

## Proposed Implementation Shape

- Likely modules or packages: add a focused `src/rphys/io/codecs.py` module for codec contracts, capabilities, datasource-neutral contexts, typed results, registry/resolution, and public codec errors. Add focused `src/rphys/data/sample_fields.py` and `src/rphys/data/sample_builders.py` modules for lazy field handles and the `IndexItem` to `Sample` bridge. Focused submodules are the canonical public import homes; package-level re-exports from `rphys.io` or `rphys.data` are allowed only when package/import tests prove they stay lightweight and do not introduce import cycles. Keep root `rphys` empty.
- Likely public classes/functions/protocols: `FieldCodec`, `CodecRegistry`, `CodecCapabilities`, `CodecLoadResult`, `CodecSaveResult`, `CodecProbeResult`, `IOContext`, `LoadContext`, `SaveContext`, `SampleField`, `SampleFieldState`, `SampleBuilder`, `SampleBuildContext`, `SampleBuildMode` or equivalent narrow helpers, and `MetadataSavePolicy`. `CodecProbeResult` is proposed even though the roadmap names only load/save results because probe needs a typed lightweight result to satisfy FR-3 without overloading `CodecLoadResult`. `MetadataSavePolicy` should be a small save-context enum, not a metadata handler, and it must be exercised by synthetic save tests before it is exported.
- Likely internal helpers: private coercion helpers for locators, requested-locator normalization, codec-operation labels, candidate sorting, capability matching, result validation, and conversion from a `CodecLoadResult` to a loaded `FieldValue`. Keep synthetic codec test helpers under tests or private fixtures, not public modules.
- Codec handler shape: `FieldCodec` should be a small structural protocol or documented duck-typed handler interface with `capabilities`, `probe(field_view, context)`, `load(field_view, context)`, `save(value, context)`, and optional pure `supports(field_view, operation, context)` behavior. Do not require downstream codecs to inherit from a stateful base class; do not expose public handler base classes, plugin hooks, or lifecycle callbacks before repeated concrete codec needs exist. `CodecRegistry` is an explicit handler selector that validates the operation methods it needs and resolves exactly one codec per operation using deterministic capability/support checks.
- Context/result shape: `IOContext`, `LoadContext`, and `SaveContext` are codec-level, datasource-neutral records. They may carry operation/source labels, `FieldLocator` as opaque caller provenance if needed, and codec policy values, but they must not import or own `RecordRef`, `IndexItem`, datasource splits, index manifests, or cache identity. `SaveContext.target` should be an explicit `FieldRef` whose ordered resources describe the output target. `SaveContext.metadata_policy` should default to `MetadataSavePolicy.REFERENCE_ONLY`, meaning codec save writes the logical field and returns provenance without persisting manifest/context metadata; `MetadataSavePolicy.INCLUDE_FIELD_METADATA` may request explicit field-metadata persistence where a codec supports it. No Stage 4 metadata policy writes datasource manifests, export reports, derived refs, layout/idempotency records, or load-time metadata. Results return loaded `FieldValue` for load, lightweight `FieldSpec` or metadata for probe, and target `FieldRef` plus written resource metadata for save.
- Lazy sample shape: `SampleField` is a `FieldValue`-compatible lazy field object accepted by `Sample.set_field()` and returned by `Sample.field()`. It must be stored as the field object, not wrapped as the payload of another `FieldValue`. `Sample.field(locator)` without validation returns the stored handle without loading; `Sample.require()`, `Sample.get()`, `Sample.field(..., expected_type=...)`, `SampleContract.validate()`, collation, and any direct `.payload` access may trigger loading because they need payload semantics. The `SampleField` handle should remain stored after success or failure so state, load result, and error remain inspectable without mutating Stage 3 descriptors. Loader callables/resolver adapters are private implementation details, not public handler interfaces.
- Sample builder shape: `SampleBuilder` consumes one `IndexItem`, optional requested locators, a `CodecRegistry`, and a `SampleBuildContext`; it builds all fields by default, supports subset and one-field helpers, supports a probe path without full payload loading, and supports explicit eager loading by touching each lazy field during build. Missing requested locators and resolution failures are typed errors. `SampleBuildContext` is the bridge-layer owner of datasource provenance such as `RecordRef` and item metadata; it projects the provenance needed by `SampleField` and results without pushing datasource objects into `rphys.io.codecs`.
- Data flow: `IndexItem.fields` plus requested locators -> normalize and validate locators -> create per-field builder/sample provenance with locator, `FieldView`, `RecordRef`, and item metadata plus a datasource-neutral codec context -> resolve codec lazily or during eager/probe paths -> store `SampleField`s in a runtime `Sample` -> `Sample.require()` or `Sample.field().payload` materializes a `FieldValue` through `FieldCodec.load()` -> save paths call `FieldCodec.save()` with a loaded field value or payload and explicit `SaveContext`.
- Dependency direction: `rphys.io.codecs` may depend on `rphys.data` vocabulary/value wrappers, `rphys.io` descriptors, and broad errors, but not on `rphys.datasources`, `rphys.ops`, `rphys.training`, or heavy optional libraries. `rphys.data.sample_fields` should avoid importing `rphys.datasources` so runtime samples stay a lightweight layer. `rphys.data.sample_builders` is the explicit bridge that may consume `rphys.datasources.IndexItem`, preferably with local or type-checking-only imports if needed to preserve package import boundaries. Stage 3 descriptor modules must not import codec or sample-builder modules.
- Extension points: downstream projects can pass explicit codec objects or registries, implement the structural `FieldCodec` interface without mandatory subclassing, provide custom codec support predicates, and wrap `SampleBuilder` without editing descriptor internals. Later export/datasource stages can reuse `CodecRegistry`, `SaveContext`, `CodecSaveResult`, and `SampleBuilder` without owning hidden global state.
- Compatibility constraints: no root `rphys` convenience exports; no plugin discovery or process-global mutable default registry in Stage 4; no descriptor mutation; no hidden full-load fallback for unsupported indexed loads; no implicit cross-sample cache; no datasource scanning, split choice, ops, model formatting, device movement, export orchestration, or training adapter behavior.
