# Codex Handoff Templates

These templates define durable Markdown artifacts for the rphys
artifact-centered Codex workflow. They complement `.codex/workflows/` and
`.codex/prompts/`: workflow entrypoints tell users how to start common
workflows, prompts tell an agent what to do, and templates define the document
shape that one or more agents draft, refine, and pass to the next stage.

Each major workflow stage leaves one durable artifact. Multiple agents or prompt
passes may work on the same artifact, but the artifact is the stage boundary.
Per-phase artifacts live under `docs/roadmap/stage-<N>/phases/`.

## Template Map

| Workflow stage or handoff | Template | Typical destination |
| --- | --- | --- |
| Roadmap-stage planning | `roadmap-stage-planning.md` | `docs/roadmap/stage-<N>/planning.md` |
| Roadmap-stage implementation plan | `roadmap-stage-implementation-plan.md` | `docs/roadmap/stage-<N>/implementation-plan.md` |
| Manager to planner | `phase-assignment.md` | manager message or `docs/roadmap/stage-<N>/phases/<phase-slug>-assignment.md` |
| Plan reviewer to manager/refiner | `plan-review-report.md` | manager thread or implementation-plan notes |
| Plan refiner to confirmation reviewer | `plan-refinement-summary.md` | manager thread or implementation-plan notes |
| Phase execution plan draft/refine | `phase-execution-plan.md` | `docs/roadmap/stage-<N>/phases/<phase-slug>.md` |
| Executor to refiner | `phase-implementation-handoff.md` | phase execution plan completion notes |
| Refiner to PR preparer | `phase-refinement-report.md` | phase execution plan completion notes |
| PR preparer to reviewer | `phase-pr-body.md` | `docs/roadmap/stage-<N>/phases/<phase-slug>-pr-body.md` |
| PR reviewer to manager | `phase-pr-review-report.md` | PR review comment or manager thread |
| Manager after merge | `phase-merge-record.md` | implementation-plan completion metadata |

`build/test-summary.md` is intentionally not templated. It is generated by
`make test-summary` and summarized into the PR body with compact Markdown
tables.
